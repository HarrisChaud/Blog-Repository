This week I got a chance to learn about the best practices that are used in modern javascript. Javascript allows the user to declare variables in many different ways, but not every way may be valid compared to others for example using var to declare variables. This way of declaring variables have been used since the inception of javascript, but it comes with many setbacks such as not being able to have block scored definitions. Since the variables defined by var are not block-scoped, redefining them in a narrower scope affects the value of the outer scope. In modern Javacript most developers now use let and const to declare variables, these differ from var, for example you cannot reassign const variables.

Example:

let y = 10

if (true) {

let y = 15 // inner declaration is scoped within the if block

console.log(y) // prints 15

}

Javascript also allows the user to create arrow functions, which have advantages over regular functions. One of the advanatages of arrow functions is that they exist within the parent functions, this helps us by removing a lot of conflicts when we are using the this keyword. for a single expression inside the arrow functions it's implied we don't have scope brackets {} and no return keyword

Example:

let y = [2, 4, 6, 8]

y.map(val => val * 2) //[4, 8, 12, 16]

Javascript provides us with the null-ish coalescing operator ?? if you want to assign a value to a variable if and only if the value is currently null-ish. A logical way to write this would be:

Example:

if (x === null) {

x = y

}

Instead we can do this:

// x = y if x is nullish, else no effect

x ??= y

This operator along with the logical AND &&= and logical OR ||= make our code a lot easier and cleaner to look at. Learning these new javascript techniques makes coding easier in javascript.
